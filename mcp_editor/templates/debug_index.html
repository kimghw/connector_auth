Debugging Workflow

Recommended Steps:

  1. Restart Server: Stop and restart the Python server
  2. Test API: Visit /api/tools to verify data
  3. Simple Test: Visit /simple-test to test basic rendering
  4. Main Editor: Visit / and open browser console (F12)
  5. Check Logs: Look for [DEBUG] messages in console
  6. Identify Issue: See where the [DEBUG] sequence stops
  7. Use Appropriate Tool: Use debug pages to investigate further

Expected Behavior

When Working Correctly:

  • Tool "get_email_handler" appears in left sidebar
  • No errors in browser console
  • Console shows complete [DEBUG] message sequence
  • Clicking tool shows editor on the right side
  • Green "Tools loaded successfully" notification appears

Common Issues

No [DEBUG] messages in console
Fix: Hard refresh browser (Ctrl+F5) - old cached version
API returns error
Fix: Restart server, check tool_definitions.py exists
Tools load but not visible
Fix: Check Elements tab in DevTools, inspect CSS styles

Browser Console Commands

Useful commands to run in browser console (F12):

// Check if tools loaded
console.log('Tools:', tools);

// Check DOM element
console.log('Element:', document.getElementById('toolList'));

// Manual test
fetch('/api/tools').then(r => r.json()).then(d => console.log(d));

// Check element visibility
const el = document.getElementById('toolList');
console.log('Display:', getComputedStyle(el).display);

Need Help?

If you're still experiencing issues after trying all debug tools:

  1. Copy all [DEBUG] and [ERROR] messages from browser console
  2. Take screenshot of Network tab showing /api/tools request
  3. Note which debug pages work vs. fail
  4. Include browser name and version
  5. Check server terminal for Python errors
